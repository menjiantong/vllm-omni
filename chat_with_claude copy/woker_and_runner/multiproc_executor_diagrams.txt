八、关键要点总结
================================================================================

1. MessageQueue 是核心:
   - 基于 共享内存环形缓冲区 + ZeroMQ 的高性能 IPC
   - 支持多 reader 的广播模式
   - Handle 用于跨进程共享连接信息

2. Pipe 仅用于初始化:
   - 在 MessageQueue 就绪前提供同步机制
   - 初始化完成后关闭

3. 只有 rank 0 返回结果:
   - 所有 Worker 执行计算
   - 只有 rank 0 聚合结果并返回
   - result_mq 的 Writer 只在 rank 0

4. BackgroundResources 确保清理:
   - weakref.finalize 机制
   - 即使异常退出也能清理资源

5. worker_monitor 监控进程:
   - 独立线程监控进程存活
   - 进程意外死亡时触发清理和回调
