# MultiprocDiffusionExecutor 通信机制详解

## 概述

`MultiprocDiffusionExecutor` 是一个多进程扩散模型执行器，它使用多种 IPC（进程间通信）机制来协调主进程与多个 GPU Worker 进程之间的通信。

---

## 一、MessageQueue 机制详解

### 1.1 核心概念

`MessageQueue` 是 vLLM 提供的高性能进程间通信机制，基于**共享内存环形缓冲区 + ZeroMQ** 实现。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          MessageQueue 架构                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Writer (主进程)                                                           │
│   ┌──────────────────┐                                                      │
│   │  enqueue()       │                                                      │
│   │      │           │                                                      │
│   │      ▼           │                                                      │
│   │  ┌────────────────────────────────────┐                                │
│   │  │    ShmRingBuffer (共享内存)         │                                │
│   │  │  ┌─────┬─────┬─────┬─────┬─────┐   │                                │
│   │  │  │chunk│chunk│chunk│ ... │chunk│   │   ← 数据区                      │
│   │  │  │  0  │  1  │  2  │     │ N-1 │   │                                │
│   │  │  └─────┴─────┴─────┴─────┴─────┘   │                                │
│   │  │  ┌─────┬─────┬─────┬─────┬─────┐   │                                │
│   │  │  │meta │meta │meta │ ... │meta │   │   ← 元数据区                    │
│   │  │  │  0  │  1  │  2  │     │ N-1 │   │                                │
│   │  │  └─────┴─────┴─────┴─────┴─────┘   │                                │
│   │  └────────────────────────────────────┘                                │
│   │      │           │                                                     │
│   │      │ notify    │ overflow → ZMQ PUB/SUB                             │
│   │      ▼           ▼                                                     │
│   │  ┌─────────┐  ┌──────────────┐                                         │
│   │  │ZMQ PUB  │  │ ZMQ PUB/SUB  │  (大数据 > 24MB)                        │
│   │  └────┬────┘  └──────┬───────┘                                         │
│   └───────┼──────────────┼─────────────────┘                               │
│           │              │                                                  │
│           │  ZMQ Socket  │                                                  │
│           │              │                                                  │
│   ┌───────▼──────────────▼─────────────────┐                               │
│   │  Reader 0    Reader 1    Reader N      │  ← 多个 Worker 进程           │
│   │  dequeue()   dequeue()   dequeue()     │                               │
│   └────────────────────────────────────────┘                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 共享内存环形缓冲区 (ShmRingBuffer)

#### 内存布局

```
内存布局:
┌─────────────────────────────────────────────────────────────────────────────┐
│                        Shared Memory Buffer                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Data Section (max_chunks × max_chunk_bytes)                              │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │ chunk0  │ chunk1  │ chunk2  │ ...  │ chunk(N-1)                     │  │
│   │ 24MB    │ 24MB    │ 24MB    │      │ 24MB                           │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   Metadata Section (max_chunks × (1 + n_reader) bytes)                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │ meta0   │ meta1   │ meta2   │ ...  │ meta(N-1)                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   每个 metadata 结构:                                                       │
│   ┌──────────────┬──────────────┬──────────────┬─────┬──────────────┐     │
│   │ written_flag │ reader0_flag │ reader1_flag │ ... │ readerN_flag │     │
│   │   (1 byte)   │   (1 byte)   │   (1 byte)   │     │   (1 byte)   │     │
│   └──────────────┴──────────────┴──────────────┴─────┴──────────────┘     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 状态转换

```
Metadata 状态机:

状态 (每个 reader 有自己的 flag):

  0???...???  → 未写入，不可读，可写
       ↓
  1000...000  → 刚写入，可读，不可写 (所有 reader flags = 0)
       ↓
  1???...???  → 已被部分 reader 读取，仍可被其他 reader 读取
       ↓
  1111...111  → 所有 reader 都已读取，不可读，可写

Writer 操作流程:
  1. 找到 written_flag=0 或 read_count=n_reader 的块
  2. 设置 written_flag=0 (标记为写入中)
  3. 写入数据
  4. 重置所有 reader_flags=0
  5. 设置 written_flag=1 (标记为可读)

Reader 操作流程:
  1. 找到 written_flag=1 且 read_flag=0 的块
  2. 读取数据
  3. 设置 read_flag=1 (标记为已读)
```

### 1.3 Handle 的作用

**Handle 是进程间共享 MessageQueue 的"钥匙"**，包含了连接共享内存和 ZMQ socket 所需的所有信息。

```python
@dataclass
class Handle:
    local_reader_ranks: list[int]  # 本地 reader 的 rank 列表
    buffer_handle: tuple           # 共享内存句柄 (n_reader, max_chunk_bytes, max_chunks, shm_name)
    local_subscribe_addr: str      # ZMQ PUB/SUB 本地地址 (ipc://...)
    local_notify_addr: str         # ZMQ 通知地址 (用于 SpinCondition)
    remote_subscribe_addr: str     # ZMQ 远程地址 (tcp://...)
    remote_addr_ipv6: bool         # 是否使用 IPv6
```

#### Handle 传递流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          Handle 传递机制                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   1. Writer (主进程) 创建 MessageQueue                                      │
│      ┌──────────────────────────────────────────┐                          │
│      │ mq = MessageQueue(n_reader=num_workers)  │                          │
│      │ handle = mq.export_handle()              │                          │
│      └──────────────────────────────────────────┘                          │
│                          │                                                  │
│                          ▼                                                  │
│   2. 通过 mp.Pipe 或 pickle 传递 handle 给 Worker 进程                      │
│      ┌──────────────────────────────────────────┐                          │
│      │ handle 被序列化并包含:                    │                          │
│      │   - 共享内存名称 (shm_name)              │                          │
│      │   - ZMQ socket 地址                      │                          │
│      │   - 配置参数                             │                          │
│      └──────────────────────────────────────────┘                          │
│                          │                                                  │
│                          ▼                                                  │
│   3. Reader (Worker 进程) 从 handle 创建 MessageQueue                       │
│      ┌──────────────────────────────────────────┐                          │
│      │ mq = MessageQueue.create_from_handle(    │                          │
│      │     handle, rank=worker_id               │                          │
│      │ )                                        │                          │
│      │                                          │                          │
│      │ # 内部操作:                              │                          │
│      │ # 1. 打开同名共享内存                    │                          │
│      │ # 2. 连接 ZMQ socket                     │                          │
│      │ # 3. 初始化 SpinCondition                │                          │
│      └──────────────────────────────────────────┘                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.4 MessageQueue 生命周期

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        MessageQueue 生命周期                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Phase 1: 创建 (Writer/主进程)                                             │
│   ─────────────────────────────                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  _broadcast_mq = MessageQueue(                                      │  │
│   │      n_reader=num_workers,           # 有多少个 reader              │  │
│   │      n_local_reader=num_workers,     # 本地 reader 数量             │  │
│   │      local_reader_ranks=[0,1,2,...]  # reader rank 列表             │  │
│   │  )                                                                  │  │
│   │  ↓                                                                  │  │
│   │  - 创建共享内存 (SharedMemory)                                       │  │
│   │  - 创建 ZMQ PUB socket                                              │  │
│   │  - 创建 SpinCondition (通知机制)                                     │  │
│   │  - 生成 handle                                                      │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   Phase 2: 连接 (Reader/Worker 进程)                                       │
│   ─────────────────────────────────                                        │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  mq = MessageQueue.create_from_handle(handle, rank)                 │  │
│   │  ↓                                                                  │  │
│   │  - 打开同名共享内存 (SharedMemory(name=...))                         │  │
│   │  - 连接 ZMQ SUB socket                                              │  │
│   │  - 创建 SpinCondition (等待通知)                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   Phase 3: 使用                                                            │
│   ──────────                                                                │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  Writer: mq.enqueue(request)    # 发送请求                          │  │
│   │  Reader: msg = mq.dequeue()     # 接收请求                          │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   Phase 4: 销毁                                                            │
│   ──────────                                                                │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  # 通过 weakref.finalize 自动清理                                   │  │
│   │  - 发送 SHUTDOWN_MESSAGE                                            │  │
│   │  - 关闭 ZMQ sockets                                                 │  │
│   │  - 关闭共享内存 (close)                                              │  │
│   │  - Writer: unlink 共享内存 (删除)                                    │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 二、mp.Pipe 管道机制

### 2.1 作用与特点

`mp.Pipe(duplex=False)` 创建的是**单向管道**，在 `MultiprocDiffusionExecutor` 中**仅用于初始化阶段的同步**。

```python
reader, writer = mp.Pipe(duplex=False)
```

- `duplex=False`: 单向管道，只能从 writer 写入，从 reader 读取
- 基于 OS 管道 (Unix: pipe, Windows: CreatePipe)
- 用于父进程与子进程之间的**一次性同步通信**

### 2.2 在初始化中的使用

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    mp.Pipe 在 Worker 初始化中的作用                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   主进程 (Executor)                                                         │
│   ┌──────────────────────────────────────────────────────────────────────┐ │
│   │  # 为每个 Worker 创建管道                                            │ │
│   │  for i in range(num_gpus):                                           │ │
│   │      reader, writer = mp.Pipe(duplex=False)                          │ │
│   │      scheduler_pipe_readers.append(reader)    # 主进程保留 reader    │ │
│   │      scheduler_pipe_writers.append(writer)    # 传给子进程           │ │
│   │                                                                       │ │
│   │      process = mp.Process(                                           │ │
│   │          target=WorkerProc.worker_main,                              │ │
│   │          args=(..., writer, ...)    # writer 传给子进程              │ │
│   │      )                                                               │ │
│   │      process.start()                                                 │ │
│   └──────────────────────────────────────────────────────────────────────┘ │
│                          │                                                  │
│                          │ 启动子进程                                       │
│                          ▼                                                  │
│   子进程 (Worker)                                                           │
│   ┌──────────────────────────────────────────────────────────────────────┐ │
│   │  def worker_main(..., pipe_writer, ...):                             │ │
│   │      # 初始化 Worker...                                              │ │
│   │      worker_proc = WorkerProc(...)                                   │ │
│   │      worker_proc.worker_busy_loop()  # 进入主循环                    │ │
│   │                                                                       │ │
│   │      # 初始化完成后，发送就绪信号                                     │ │
│   │      pipe_writer.send({                                              │ │
│   │          "status": "ready",                                          │ │
│   │          "result_handle": result_mq_handle  # 仅 rank 0 发送         │ │
│   │      })                                                              │ │
│   └──────────────────────────────────────────────────────────────────────┘ │
│                          │                                                  │
│                          │ 发送就绪消息                                     │
│                          ▼                                                  │
│   主进程 (Executor)                                                         │
│   ┌──────────────────────────────────────────────────────────────────────┐ │
│   │  # 关闭所有 writer 端 (不再需要)                                      │ │
│   │  for writer in scheduler_pipe_writers:                               │ │
│   │      writer.close()                                                  │ │
│   │                                                                       │ │
│   │  # 等待所有 Worker 就绪                                               │ │
│   │  for reader in scheduler_pipe_readers:                               │ │
│   │      data = reader.recv()                                            │ │
│   │      if data["status"] != "ready":                                   │ │
│   │          raise RuntimeError("Init failed")                           │ │
│   │      result_handle = data.get("result_handle")  # 从 rank 0 获取     │ │
│   │      reader.close()  # 关闭管道                                      │ │
│   └──────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
│   注意: Pipe 在初始化完成后就被关闭，不再使用                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.3 为什么需要 Pipe？

| 阶段 | 使用 Pipe 的原因 |
|------|-----------------|
| Worker 启动前 | MessageQueue 还未建立连接 |
| 初始化中 | 需要同步等待 Worker 就绪 |
| 获取 result_handle | rank 0 需要回传 result MessageQueue 的 handle |
| 初始化完成后 | Pipe 被关闭，后续使用 MessageQueue |

## 三、完整通信架构

### 3.1 通信渠道总览

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    MultiprocDiffusionExecutor 通信架构                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│                         ┌─────────────────────┐                            │
│                         │    主进程 (Executor) │                            │
│                         │                     │                            │
│                         │  _broadcast_mq      │  ← 发送请求                │
│                         │  _result_mq         │  ← 接收结果                │
│                         └─────────┬───────────┘                            │
│                                   │                                         │
│          ┌────────────────────────┼────────────────────────┐               │
│          │                        │                        │               │
│          ▼                        ▼                        ▼               │
│   ┌─────────────┐          ┌─────────────┐          ┌─────────────┐       │
│   │  Worker 0   │          │  Worker 1   │          │  Worker N   │       │
│   │  (rank 0)   │          │  (rank 1)   │          │  (rank N)   │       │
│   │             │          │             │          │             │       │
│   │ mq (reader) │          │ mq (reader) │          │ mq (reader) │       │
│   │ result_mq   │          │    -        │          │    -        │       │
│   │  (writer)   │          │             │          │             │       │
│   └─────────────┘          └─────────────┘          └─────────────┘       │
│                                                                             │
│   通信渠道:                                                                 │
│   ┌─────────────────────────────────────────────────────────────────────┐ │
│   │ 1. _broadcast_mq (主进程 → 所有 Worker)                             │ │
│   │    - 共享内存环形缓冲区                                              │ │
│   │    - ZMQ PUB/SUB (大数据)                                            │ │
│   │    - 用于: 发送 RPC 请求、生成请求、shutdown 信号                    │ │
│   │                                                                      │ │
│   │ 2. _result_mq (Worker 0 → 主进程)                                   │ │
│   │    - 共享内存环形缓冲区                                              │ │
│   │    - 用于: 返回执行结果                                              │ │
│   │    - 只有 rank 0 拥有 result_mq 的 writer 端                        │ │
│   │                                                                      │ │
│   │ 3. mp.Pipe (初始化阶段)                                              │ │
│   │    - OS 管道                                                         │ │
│   │    - 用于: 同步 Worker 就绪状态、传递 result_handle                 │ │
│   │    - 初始化完成后关闭                                                │ │
│   │                                                                      │ │
│   │ 4. mp.Event (wake_event)                                            │ │
│   │    - 跨进程事件                                                      │ │
│   │    - 用于: 唤醒处于 sleep 状态的 Worker                             │ │
│   └─────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 Worker 进程启动流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Worker 进程启动流程                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   主进程                            Worker 进程                             │
│   ────────                          ────────────                            │
│                                                                             │
│   创建 Pipe:                                                                │
│   reader, writer = mp.Pipe()                                               │
│                                                                             │
│   启动进程:                          WorkerProc.worker_main():              │
│   process = mp.Process(            │                                       │
│       target=worker_main,          │   1. 初始化 GPU 环境                   │
│       args=(rank, od_config,       │      └── set_device(rank)             │

## 五、关键设计要点

### 5.1 为什么用两种通信机制？

| 机制 | Pipe | MessageQueue |
|------|------|--------------|
| **时机** | 仅初始化阶段 | 整个运行时 |
| **方向** | 单向 (Worker → 主进程) | 双向 (广播/结果) |
| **用途** | 同步就绪状态 | 请求/结果传递 |
| **原因** | 初始化时 MessageQueue 未就绪 | 高性能共享内存通信 |

### 5.2 为什么只有 rank 0 返回结果？

### 5.3 SpinCondition 机制

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        SpinCondition 等待机制                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   问题: 如何高效等待共享内存中的数据？                                       │

## 六、总结

### 通信机制对比

| 组件 | 技术 | 用途 | 生命周期 |
|------|------|------|----------|
| `broadcast_mq` | 共享内存 + ZMQ | 主进程 → 所有 Worker | 运行时 |
| `result_mq` | 共享内存 + ZMQ | Worker 0 → 主进程 | 运行时 |
| `mp.Pipe` | OS 管道 | 初始化同步 | 初始化阶段 |
| `mp.Event` | 进程间事件 | 唤醒 sleep 的 Worker | 运行时 |

### 关键点

1. **MessageQueue** 是高性能共享内存通信的核心，支持多 reader 的广播模式
2. **Handle** 是跨进程共享 MessageQueue 的关键，包含所有连接信息
3. **Pipe** 仅用于初始化阶段的同步，之后即关闭
4. **只有 rank 0** 拥有 result_mq 的写入权，负责返回最终结果
5. **BackgroundResources** 确保资源正确清理，即使异常退出
