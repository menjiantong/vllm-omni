三、两种执行模式对比
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│                    execute_model (一次性完整推理)                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   输入: OmniDiffusionRequest                                                │
│   输出: DiffusionOutput                                                     │
│                                                                             │
│   时间线:                                                                   │
│   ──────                                                                    │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐ │
│   │                                                                     │ │
│   │   Request ──► [encode] ──► [denoise step 0] ──► [denoise step 1]   │ │
│   │                                                   │                 │ │
│   │                                                   ▼                 │ │
│   │                              [denoise step N] ──► [decode] ──► Output│
│   │                                                                     │ │
│   │   所有步骤在 pipeline.forward() 内部完成                            │ │
│   └─────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
│   特点:                                                                     │
│   - 简单直接                                                                │
│   - 适合单次请求                                                            │
│   - 无法中途抢占                                                            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                    execute_stepwise (逐步可控推理)                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   输入: DiffusionSchedulerOutput                                            │
│   输出: RunnerOutput                                                        │
│                                                                             │
│   时间线:                                                                   │
│   ──────                                                                    │
│                                                                             │
│   调用 1:                                                                   │
│   ┌─────────────────────────────────────────────────────────────────────┐ │
│   │  prepare_encode(state)                                               │ │
│   │  denoise_step(state) ──► noise_pred                                  │ │
│   │  step_scheduler(state, noise_pred)                                   │ │
│   │  return RunnerOutput(finished=False)                                 │ │
│   └─────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
│   调用 2:                                                                   │
│   ┌─────────────────────────────────────────────────────────────────────┐ │
│   │  denoise_step(state) ──► noise_pred                                  │ │
│   │  step_scheduler(state, noise_pred)                                   │ │
│   │  return RunnerOutput(finished=False)                                 │ │
│   └─────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
│   ...                                                                       │
│                                                                             │
│   调用 N:                                                                   │
│   ┌─────────────────────────────────────────────────────────────────────┐ │
│   │  denoise_step(state) ──► noise_pred                                  │ │
│   │  step_scheduler(state, noise_pred)                                   │ │
│   │  post_decode(state) ──► result                                       │ │
│   │  return RunnerOutput(finished=True, result=result)                   │ │
│   └─────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
│   特点:                                                                     │
│   - 每步可控                                                                │
│   - 支持抢占和调度                                                          │
│   - 支持多请求交错                                                          │
│   - 需要管理 state_cache                                                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

传输模式:
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  1. full_payload_mode:                                                      │
│     - 完整数据一次性传输                                                     │
│     - 请求完成时发送                                                         │
│     - 适用: 非流式场景                                                       │
│                                                                             │
│  2. async_chunk:                                                            │
│     - 数据分块流式传输                                                       │
│     - 每个 step 发送一个 chunk                                              │
│     - 适用: Thinker → Talker                                                │
│                                                                             │
│  3. KV Transfer:                                                            │
│     - KV Cache 跨阶段复用                                                   │
│     - 支持异构 TP (rank-aware routing)                                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘


五、Cache-DiT 加速原理
================================================================================

问题: DiT 每步都要完整计算
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  Step 0:  Input ──► [Attention] ──► [FFN] ──► [Attention] ──► Output      │
│  Step 1:  Input ──► [Attention] ──► [FFN] ──► [Attention] ──► Output      │
│  Step 2:  Input ──► [Attention] ──► [FFN] ──► [Attention] ──► Output      │
│  ...                                                                        │
│                                                                             │
│  每步计算量相同，但中间激活变化很小！                                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

解决方案: 缓存中间激活
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  Step 0:  Input ──► [Attention] ──► [FFN] ──► [Attention] ──► Output      │
│                        │           │            │                           │
│                        ▼           ▼            ▼                           │
│                    ┌─────────────────────────────────┐                     │
│                    │      Cache Storage              │                     │
│                    │  cache_0, cache_1, cache_2      │                     │
│                    └─────────────────────────────────┘                     │
│                                                                             │
│  Step 1:  Input ──► [Cache+Δ] ──► [Cache+Δ] ──► [Cache+Δ] ──► Output      │
│                                                                             │
│  后续步骤只需计算增量 Δ，大幅减少计算量                                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

代码使用:
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  # 配置                                                                     │
│  od_config.cache_backend = "cache_dit"                                     │
│  od_config.cache_config = {...}                                            │
│                                                                             │
│  # 在 load_model 中                                                         │
│  cache_backend = get_cache_backend("cache_dit", cache_config)              │
│  cache_backend.enable(pipeline)                                             │
│                                                                             │
│  # 在 execute_model 中                                                      │
│  cache_backend.refresh(pipeline, num_inference_steps)                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

八、与 Worker 的交互
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│                     DiffusionWorker 调用 ModelRunner                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   class DiffusionWorker:                                                    │
│       def __init__(self, ...):                                              │
│           self.model_runner = DiffusionModelRunner(...)                    │
│                                                                             │
│       def load_model(self, ...):                                            │
│           self.model_runner.load_model(...)                                │
│                                                                             │
│       def execute_model(self, req, od_config):                              │
│           │                                                                 │
│           ├── 1. 设置 LoRA adapter                                         │
│           │   lora_manager.set_active_adapter(req.sampling_params.lora)    │
│           │                                                                 │
│           ├── 2. 获取 profiler context                                     │
│           │   ctx = profiler.annotate_context_manager("diffusion_forward") │
│           │                                                                 │
│           ├── 3. 调用 model_runner                                         │
│           │   with ctx:                                                    │
│           │       output = self.model_runner.execute_model(req)            │
│           │                                                                 │
│           └── 4. 更新 profiler                                             │
│               profiler.step()                                              │
│                                                                             │
│           return output                                                     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

职责分离:
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   DiffusionWorker:                  DiffusionModelRunner:                  │
│   ────────────────                  ─────────────────────                  │
│   - 设备初始化                      - 模型加载                             │
│   - 分布式环境                      - 模型编译                             │
│   - 进程管理 (sleep/wake)           - Cache Backend 管理                   │
│   - LoRA 管理                       - Offload Backend 管理                 │
│   - Profiler 管理                   - 推理执行                             │
│                                     - 跨阶段数据传输                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
